# Практический проект: Конвейер данных на Go (Горутины и Каналы)
Этот проект был создан для закрепления ключевых концепций многопоточного программирования на Go, фокусируясь на построении надежного конвейера (pipeline) обработки данных с использованием горутин, каналов и контекста.

## Цели и Задачи
Проект демонстрирует, как организовать конкурентную обработку данных в сложной архитектуре:

- Паттерн Generator-Worker-Collector: Реализована классическая схема: Источник (создает данные) → Обработчик (трансформирует данные) → Сборщик (агрегирует результат).

- Безопасное завершение: Использование context.WithTimeout для ограничения времени работы и корректной остановки всех горутин по истечении 1 секунды.

- Синхронизация: Применение sync.WaitGroup для точного ожидания завершения всех горутин-сборщиков, прежде чем закрывать результирующий канал.

- Безопасный подсчет: Использование атомарных операций (sync/atomic) для безопасного увеличения счетчиков сумм и количеств в параллельных потоках.

- Конкуренция за данные: Демонстрация того, как пять горутин-Обработчиков (Worker) конкурируют за получение чисел из одного общего канала.

## Структура Конвейера
Проект состоит из трех основных функциональных блоков, работающих параллельно:

1. Генератор (Generator)
Эта горутина является источником данных. Она генерирует последовательность чисел (1, 2, 3...) и отправляет их в канал chIn. Генератор постоянно мониторит состояние context.Done() и корректно завершается, закрывая свой канал через defer close(chIn), когда время работы истекает.

2. Обработчики (Worker)
Запущено пять горутин Worker. Они читают данные из общего входного канала chIn, имитируют задержку (time.Sleep(1 * time.Millisecond)), и передают результат в свои индивидуальные выходные каналы (outs).

3. Сборщики (Анонимные горутины)
Запущено пять горутин-сборщиков. Каждая из них читает данные из одного из каналов outs, собирает статистику (слайс amounts) и направляет все числа в один финальный канал (chOut). После завершения работы всех пяти сборщиков, специальная горутина использует wg.Wait() для закрытия результирующего канала chOut.

## Запуск и Проверка
Программа запускает конвейер, ожидает его завершения по таймауту, а затем сравнивает итоговые суммы и количество полученных чисел с количеством и суммой сгенерированных чисел.

Запуск проекта:

`go run main.go` 
 
Финальная часть программы выполняет строгую проверку:

Суммы и количества сгенерированных и полученных чисел должны совпадать.

Сумма всех чисел, собранных в пяти каналах-сборщиках (amounts), должна совпадать с общим количеством сгенерированных чисел.

Если проверка пройдена, это подтверждает, что ни одно число не было потеряно и не было обработано дважды в ходе параллельной работы конвейера.
